
# 数据操作概述

EMOP 平台提供了多种灵活的数据操作方式，以满足不同场景和用户的需求。本文将概述这些操作方式，并帮助您选择最适合的方案。

## 数据操作方式

EMOP 支持以下几种主要的数据操作方式：

### [Java API](java-api)
   - 面向后端开发人员
   - 提供最完整的功能支持
   - 支持复杂的业务逻辑处理
   - 提供强类型的编程体验
   - 细粒度的事务控制

### [DSL（领域特定语言）](dsl)
   - 面向系统实施及开发人员
   - 简洁直观的语法
   - 适合批量数据操作
   - 支持元数据和数据的统一管理
   - 方便的导入导出功能

### [REST API](rest-api)
   - 面向集成开发人员
   - 标准的 HTTP 接口
   - 支持跨平台集成
   - 适合分布式系统架构
   - 支持第三方系统集成


### [前端代码](frontend)
   - 面向前端开发人员
   - 完整的与Java API一致的API操作体验

### [XPath 数据操作](xpath)
   -   面向所有开发人员
   -   树形数据导航和操作
   -   支持复杂路径访问
   -   支持自动对象创建
   -   集成在其他操作方式(Java API, DSL, REST API, 前端代码都可以使用xpath)
   
### [全文检索](fulltext-search)
   - 支持跨对象类型的内容检索
   - 基于字段权重的相关性排序
   - 提供智能搜索建议
   - 实时索引更新能力
   
## 核心概念

在开始使用这些数据操作方式之前，需要了解以下核心概念：

1. **数据模型**
   - ModelObject：所有数据对象的基础接口
   - 支持动态属性扩展
   - 内置版本控制机制

2. **事务处理**
   - 支持 ACID 事务
   - 暂不支持分布式事务

3. **数据一致性**
   - 业务唯一键约束
   - 数据验证机制

## 通用功能

所有操作方式都支持以下基本功能：

- 基础的 CRUD 操作
- 批量数据处理
- 关系数据处理

## 最佳实践

1. **选择合适的操作方式**
   - 考虑使用场景和用户角色
   - 评估性能需求
   - 考虑维护成本

2. **数据操作原则**
   - 遵循幂等性原则
   - 注意数据一致性
   - 合理使用事务
   - 做好错误处理

3. **性能考虑**
   - 批量操作优于单条操作
   - 合理使用查询条件
   - 避免大事务
   - 注意数据量级
